Domaine Canada.ca

Tout site Web a un domaine principal qui représente son objectif principal. Le domaine Canada.ca est le domaine principal pour la présence Web publique du gouvernement du Canada.

Tous les sites Web publics du gouvernement du Canada doivent utiliser le domaine Canada.ca, à une exception près : ceux qui sont spécifiquement destinés au personnel actuel ou ancien du gouvernement du Canada peuvent utiliser le domaine .gc.ca. Pour des questions d’uniformité et de confiance, tous les autres sites destinés au public, notamment ceux où une connexion est requise, doivent utiliser le domaine Canada.ca.

Le domaine Canada.ca peut être utilisé de deux façons :

  • Sans sous-domaine, par exemple : https://www.canada.ca/fr/services/emplois.html
  • Avec un sous-domaine, par exemple : https://agriculture.canada.ca/fr/science

Les noms de domaines et de sous-domaines doivent tous être approuvés par Service Canada, et leur résolution doit être possible sans le préfixe « www. ».

Quoi éviter

Il ne faut pas utiliser le domaine Canada.ca pour les sites collaboratifs où le gouvernement du Canada n’est pas le partenaire principal. Ce type de site comprend les sites Web réalisés conjointement avec des partenaires autres que le GC, par exemple d’autres ordres de gouvernement ou des organismes non gouvernementaux, et pour lesquels le GC n’est pas l’organisation responsable.

Sous-domaines

Un sous-domaine est une subdivision du domaine principal. Il permet d’organiser des sections distinctes au sein du site Web principal. Par exemple, canada-ca.github.io/systeme-conception est un sous-domaine de canada.ca.

Limitez l’utilisation de sous-domaines uniquement aux contenus et services hébergés en dehors du Service Web géré.

Au moment de choisir un sous-domaine :

  • On recommande un maximum de 2 mots-clés.
  • Regroupez les sites secondaires et les services numériques apparentés sous le même sous-domaine, si possible.
  • Suivez les directives de la section Comment écrire une adresse URL.
    • Exemples :
      • bibliotheque-archives.canada.ca
      • inspection.canada.ca

Remarque : Les demandes de sous-domaine sont examinées par Service Canada. Votre gestionnaire de compte Web peut soumettre des demandes de nouveaux sous-domaines Canada.ca et de modification à l’Éditeur principal à l’aide du formulaire de demande pour Canada.ca (interne au gouvernement).

Traiter les deux langues officielles de manière égale

Les sites et applications Web du gouvernement du Canada doivent traiter de manière égale les deux langues officielles.

Vous pouvez répondre aux exigences linguistiques officielles pour les domaines et sous-domaines en mettant en œuvre au moins l’une des options suivantes.

Noms unilingues équivalents

Vous pouvez employer un nom unilingue équivalent à un autre nom unilingue pour chacune des langues officielles. Par exemple , jeunesse.canada.ca en français et youth.canada.ca en anglais.

La résolution du nom de domaine unilingue doit aboutir à la page d’accueil du site Web dans la langue en question. Par exemple, la résolution de sante.canada.ca aboutit à la page d’accueil en français, et la résolution de health.canada.ca aboutit à la page d’accueil en anglais.

Noms uniques bilingues

Vous pouvez employer un nom unique bilingue (compris dans chacune des langues) ou un terme composé qui représente l’objectif principal du site Web dans les deux langues officielles à la fois. Par exemple : agriculture.canada.ca ou tc.canada.ca.

La résolution du nom de domaine bilingue aboutit à la page d’entrée du site Web sauf si la préférence de langue est connue, dans quel cas la résolution aboutit à la page d’accueil dans la langue en question. Par exemple, la résolution de international.canada.ca aboutit à une page d’entrée, sauf si la préférence de langue est connue.

Modèle d’URL

Chaque page Web a une adresse URL principale qui apparaît dans la barre d’adresse du navigateur. Toute autre adresse URL renvoyant à la page Web est résolue en rendant l’URL principale. C’est le cas pour les alias d’URL, les URL personnalisées, les URL de redirection et toute autre URL qui n’est pas l’URL principale.

Comment écrire une adresse URL

Les instructions suivantes s’appliquent à toutes les adresses URL, y compris les URL personnalisées et de sous-domaine :

  • Utilisez un langage clair pour les mots-clés.
  • Séparez les mots-clés d’une même section de chemin par des traits d’union.
  • Évitez les mots superflus comme « à », « de », « le », « un », « par », « pour », etc. Par exemple, écrivez « /demande-pret-etudiant » et non « /comment-faire-une-demande-de-pret-etudiant ».
  • Utilisez des minuscules.
  • Utilisez des mots-clés unilingues, dans la langue de la page.
  • Utilisez uniquement des caractères US-ASCII (une version ASCII à 7 bits). Par exemple, utilisez « meteo » et non « météo ».
  • Évitez les répétitions de mots-clés et les apostrophes.
  • Évitez les acronymes et les abréviations, sauf s’ils sont mieux compris que la version écrite au long ou qu’ils génèrent de meilleurs résultats sur les moteurs de recherche.

Adresses URL personnalisées

Une URL personnalisée est une adresse brève et facile à comprendre qui renvoie à une page dont l’URL est longue.

Lorsque vous utilisez des adresses URL dans des produits de communication (publications, dépliants, etc.) ou des campagnes publicitaires (télévision, radio, presse, etc.), vous pouvez utiliser soit l’URL principale, soit une URL personnalisée. Par exemple, l’URL personnalisée canada.ca/pension-avantages renvoie à l’adresse https://www.canada.ca/fr/secretariat-conseil-tresor/sujets/pension-avantages-sociaux.html

Caractéristiques obligatoires d’une URL personnalisée :

  • Être facile à taper et à communiquer.
  • Comprendre une seule section de chemin (une seule barre oblique).
  • Masquer l’extension de fichier pour les pages HTML (par exemple, « canada.ca/passeport », « canada.ca/meteo »).
  • Respecter les directives de la sectione Comment écrire une adresse URL.

Si l’adresse URL doit être lue à voix haute (par exemple, dans des publicités à la radio ou à la télévision), vous pouvez créer une URL personnalisée supplémentaire sans traits d’union. La lecture à voix haute s’en trouvera facilitée.

Exemple — Publicité à la télévision :

  • URL personnalisée pour les imprimés : canada.ca/assurance-emploi-maternite
  • URL personnalisée pour la télévision ou la radio : canada.ca/assuranceemploimaternite (prononcée « canada point c a barre oblique assurance emploi maternité »)

Remarque : Votre gestionnaire de compte Web peut soumettre des demandes de nouvelle URL personnalisée et de modification à l’Éditeur principal à l’aide du formulaire de demande pour Canada.ca (interne au gouvernement).

Pour en savoir plus sur les URL personnalisées, consultez :

  • Adresses URL personnalisées pour Canada.ca — GCéchange (interne au gouvernement)
